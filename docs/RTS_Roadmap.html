Phase 1: Unit Movement 1-2 tuần

#TaskMô tả
1.1NavigationRegion2DThêm NavigationRegion2D vào node Navigation, bake navigation polygon trên vùng đất đi được
1.2NavigationAgent2DThêm NavigationAgent2D vào BaseUnit.tscn
1.3Right-click di chuyểnSelectionManager xử lý Right-click → gọi unit.MoveTo(position)
1.4Unit State MachineEnum UnitState { Idle, Moving, Attacking, Gathering, Building, Dead } + switch trong _PhysicsProcess
1.5Walk animationThêm animation "walk" cho Pawn (sprite sheet đã có sẵn)
1.6MoveAndSlideBaseUnit.MoveTo() set target → state = Moving → NavigationAgent tính path → Velocity + MoveAndSlide()

Phase 2: Selection nâng cao 1 tuần

#TaskMô tả

Phase 3: Resource System 2 tuần

#TaskMô tả
3.1ResourceManager.csSingleton quản lý: Gold, Wood, Food, Stone cho mỗi player. Signal khi thay đổi
3.2ResourceNode.csBase class cho tài nguyên trên map: TreeNode, GoldMine, StoneMine, BerryBush
3.3Resource scenesTạo scene từ assets có sẵn: Trees, Gold Mine (Assets/Tiny Swords Update 010/Resources/)
3.4Gather logicPawn right-click resource → Walk tới → Gather (timer) → Walk về TownHall → Deposit → Loop
3.5Carry & depositPawn mang CarryCapacity → giao cho ResourceManager khi về gần building
3.6Resource depletionTài nguyên hết → animation biến mất → QueueFree()
3.7HUD resource barUI hiển thị Gold/Wood/Food/Stone trên cùng (trong node UILayer)

Phase 4: Building System 2-3 tuần

#TaskMô tả

📚 Kiến thức cần nắm

  • Godot Navigation2D — pathfinding cho unit di chuyển
  • Godot TileMap — terrain autotiling, physics layers
  • C# Signals / Events — giao tiếp giữa các node loosely-coupled
  • State Machine pattern — FSM quản lý trạng thái unit (Idle/Moving/Attack...)
  • Godot UI (Control nodes) — xây dựng HUD, panels, buttons
  • Shader cơ bản — cho Fog of War, selection highlight, damage flash
  • Godot Resource (.tres) — data-driven design cho unit stats, building costs
